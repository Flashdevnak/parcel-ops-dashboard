## เปิดทดสอบบนเครื่องแบบไม่ติด cache
แนะนำใช้ไฟล์นี้บน Windows:

```bat
START_PRODUCTION_PREVIEW.bat
```

หรือใช้คำสั่ง:

```bash
python dev-server.py
```

แล้วเปิด:

```text
http://localhost:5173
```

## Git Ready Final
- ชุดนี้เป็นตัวพร้อมขึ้น Git / GitHub Pages / Vercel
- ไม่มี `?v=xx` ต่อท้าย URL
- เมนูทุกหน้าเป็น URL สะอาด
- คัดลอกไลน์งานรองรับ:
  1. ภาพรวม: สรุปตาม HUB ปลายทาง
  2. สถานะและกราฟ: สถานะดำเนินการล่าสุด / FD / LH
  3. FD / สาขา: สรุปตามสาขาปลายทาง
  4. LH / HUB อื่น: สรุปตาม HUB ปลายทางอื่น
  5. รายการพัสดุ: สรุปตาม HUB ปลายทาง + จำนวนชิ้น + ค้างนานสุด + เลขพัสดุตัวอย่างแยกตามช่วงเวลา
- หน้า `รายการพัสดุ` จัดหมวดเลขพัสดุตัวอย่างตามระยะเวลาในคลัง:
  `≤6 ชม.`, `≤12 ชม.`, `≤22 ชม.`, `22–24 ชม.`, `>24 ชม.`, `>48 ชม.`
- ใช้รูปแบบ `ช่วงเวลา: จำนวน` และคั่นด้วย `|`
- กราฟสถานะ / FD / LH มี label ตัวเลขบนกราฟ
- ตรวจสอบด้วย:
  - `node --check app.bundle.js`
  - `python -m py_compile dev-server.py`
  - static link / asset checks
